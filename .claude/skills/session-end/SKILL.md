# Skill: Session End

> Завершение рабочей сессии. Обновляет memory, логирует ошибки, оценивает качество.
> Запуск: `/session-end` или "заверши сессию", "конец работы"

---

## Когда вызывать

- В конце рабочей сессии
- Перед длительным перерывом
- При переключении на другой проект

---

## Процесс

### 1. Обзор сессии

Ответить на 5 вопросов:

```markdown
## Session Review — {date}

### Что сделано:
1. {task 1}
2. {task 2}
3. {task 3}

### Ключевые решения:
| Решение | Причина |
|---------|---------|
| {что решили} | {почему} |

### Ошибки/проблемы:
| Ошибка | Root Cause | Урок |
|--------|-----------|------|
| {что} | {почему} | {что запомнить} |

### Новые правила (если обнаружены):
- {правило 1}
```

### 2. Обновить experience.md

Если были ошибки — добавить в `experience.md` (проектный):

```markdown
| # | Date | Type | Error | Root Cause | Lesson | Prevention |
|---|------|------|-------|-----------|---------|------------|
| N | {date} | {type} | {error} | {cause} | {lesson} | {prevention} |
```

Типы: Build, Runtime, Logic, Test, Deploy, Config, API, Type

### 3. Обновить MEMORY.md

В `~/.claude/projects/<hash>/memory/MEMORY.md`:
- Обновить "Current State" для проекта
- Добавить ключевые решения в таблицу
- Обновить Tech Debt / TODO если нужно

### 4. Обновить AGENTS.md (если есть)

Если структура проекта изменилась — обновить:
- Новые файлы / команды
- Изменённую архитектуру
- Версию и дату

### 5. Оценить качество сессии

```markdown
## Quality Score

| Критерий | Оценка (1-10) |
|----------|--------------|
| Accuracy — правильность решений | {X} |
| Completeness — всё ли доделано | {X} |
| Quality — чистота кода/артефактов | {X} |
| Process — следование правилам | {X} |
| Learning — чему научились | {X} |
| **Средний балл** | **{X.X}** |

### Комментарий:
{что можно улучшить в следующей сессии}
```

### 6. Предложить следующие шаги

```markdown
### Next session:
1. {конкретная задача}
2. {конкретная задача}

### Hypothesis:
{гипотеза о том, что будет важно в следующей сессии}
```

### 7. Записать в experience.md (если нужно)

Если quality score < 7 по любому критерию — записать в experience.md как pattern.

---

## Правила

- НЕ пропускать оценку качества — она нужна для трендов
- НЕ придумывать проблемы — если сессия прошла хорошо, так и написать
- НЕ обновлять корневой CLAUDE.md — только проектные файлы и memory
- Обновлять MEMORY.md кратко — он загружается каждую сессию, max 200 строк
