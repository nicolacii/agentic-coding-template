# Skill: Session Start

> Инициализация рабочей сессии. Читает контекст, показывает статус, предлагает следующие шаги.
> Запуск: `/start` или начало новой сессии

---

## Когда вызывать

- В начале каждой рабочей сессии
- После длительного перерыва
- При переключении между проектами

---

## Процесс

### 1. Прочитать контекст

**Обязательно:**
- `CLAUDE.md` (текущего проекта, если есть)
- `AGENTS.md` (архитектура проекта)
- Memory files (MEMORY.md — загружается автоматически)

**Если проект с pipeline:**
- `WORKFLOW.md` — текущий процесс
- `tasks/reflection-history.md` — история рефлексий
- Последний незавершённый `tasks/{section}/` — где остановились

**Если проект с legacy/миграцией:**
- `RESEARCH.md` — только оглавление, не весь файл

### 2. Проверить статус проекта

```bash
# Git status
git status

# Незавершённые задачи
find tasks/ -name "tasks-migrate-*.md" -exec grep -l "^\- \[ \]" {} \;

# Последние коммиты
git log --oneline -5

# Сервер/тесты
npx tsc --noEmit 2>&1 | tail -3
```

### 3. Проверить experience.md

Прочитать `experience.md` (если существует) — последние ошибки и паттерны.
Есть ли ошибки которые повторяются? Нужно ли обновить правила?

### 4. Показать сводку

```markdown
## Session Start — {date}

### Проект: {name}
**Статус:** {описание текущего состояния}

### Git:
- Branch: {branch}
- Uncommitted: {N files}
- Last commit: {message} ({time ago})

### Pipeline (если есть):
- Текущий раздел: {section}
- Текущий этап: {1-6}
- Pending tasks: {N}

### Незавершённое:
- [ ] {task 1}
- [ ] {task 2}

### Recurring errors (из experience.md):
- {pattern 1} — {N раз}

### Предложение:
Продолжить с {что делать дальше}. Или: {альтернатива}.
```

### 5. Ждать указаний пользователя

НЕ начинать работу без указания. Показать сводку и ждать.

---

## Правила

- НЕ читать RESEARCH.md целиком (слишком большой) — только оглавление
- НЕ запускать тяжёлые команды (build, test) — только status checks
- НЕ предлагать больше 2-3 следующих шагов
- Сводка должна быть < 30 строк
