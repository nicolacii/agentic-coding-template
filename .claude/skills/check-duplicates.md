# Skill: Check Duplicates

> Проверка на дублирование перед созданием нового файла/функции/компонента.
> Запуск: `/check-duplicates` или "проверь дубли", "есть ли такое уже"

---

## Когда вызывать

**ПЕРЕД созданием ЛЮБОГО нового:**
- Файла, класса, функции, компонента, хука, MCP сервера

---

## Процесс

### 1. Поиск существующих реализаций

```
1. Semantic search: "Как реализован [функционал]?"
2. Grep: по ключевым терминам, именам функций/классов
3. Glob: по похожим именам файлов
4. Проверить: lib/, utils/, shared/, components/, hooks/, services/, modules/
```

### 2. Отчёт

```markdown
## DUPLICATE CHECK REPORT

### Target: [что хочу создать]
- Type: [file/class/function/component/hook]
- Name: [предложенное имя]
- Purpose: [что делает]

### Search Results:
1. Semantic: [found/not found] — [детали]
2. Grep: [found/not found] — [детали]
3. Glob: [found/not found] — [детали]

### Similar Found:
- [ ] [path] — [описание сходства]

### Decision:
- [ ] CREATE NEW — дублей не найдено (confidence >= 0.7)
- [ ] EXTEND EXISTING — найден аналог в [path]
- [ ] ASK USER — найден частичный аналог
```

### 3. Матрица решений

| Confidence | Действие |
|------------|----------|
| >= 0.9 | Создать без вопросов |
| 0.7–0.9 | Создать, упомянуть в чате |
| 0.5–0.7 | Спросить пользователя |
| < 0.5 | НЕ СОЗДАВАТЬ — найти существующее |

---

## Запреты

- НИКОГДА: файлы с `_fixed`, `_final`, `_new`, `_v2`
- НИКОГДА: пропускать проверку "потому что уверен"
- НИКОГДА: создавать утилиту без проверки utils/, lib/, shared/
