# Skill: Backlog to Rules

> Внедрение накопленных улучшений из backlog/рефлексий в правила проекта.
> Запуск: `/backlog-to-rules` или "внедри улучшения", "обнови правила из backlog"

---

## Когда вызывать

| Условие | Приоритет |
|---------|-----------|
| 5+ улучшений накопилось | Обязательно |
| Прошла неделя с последнего внедрения | Рекомендуется |
| 2+ High priority улучшения | Обязательно |
| Ошибка повторилась 3+ раз | Немедленно |

---

## Процесс (7 фаз)

### Phase 1: ANALYZE

1. Прочитать источники улучшений:
   - `tasks/reflection-history.md` — рефлексии по циклам
   - `tasks/*/reflection.md` — рефлексии по разделам
   - Error logs, review findings

2. Сгруппировать по целевому файлу:
```markdown
| Файл | Улучшения | Кол-во |
|------|-----------|--------|
| CLAUDE.md | #X, #Y | N |
| WORKFLOW.md | #Z | M |
```

### Phase 2: RESEARCH

Для каждого целевого файла:
- Прочитать текущую структуру
- Определить куда добавить (секция / новая секция)
- Проверить конфликты с существующими правилами

### Phase 3: PLAN

```markdown
## ПЛАН ВНЕДРЕНИЯ
### Файл: [path]
- [ ] Добавить: [что]
- [ ] Изменить: [что]
### Критерии: все улучшения внедрены, контекст сохранён
```

### Phase 4: IMPLEMENT

Правила внедрения:
- Добавлять новые секции, НЕ заменять существующие
- Включать конкретный пример
- Добавлять **WHY** с реальным случаем
- Сохранять формат файла

### Phase 5: VERIFY

- Открыть каждый изменённый файл
- Проверить структуру
- Убедиться что связи не нарушены

### Phase 6: UPDATE SOURCES

Обновить статусы улучшений → Done с датой.

### Phase 7: SUMMARY

```markdown
## IMPLEMENTATION SUMMARY
Внедрено: N улучшений
Обновлено: N файлов
Паттерны предотвращены: [список]
```

---

## Запреты

- НЕ удалять существующие секции
- НЕ объединять несвязанные правила
- НЕ писать абстрактно — только конкретные примеры
- НЕ менять нумерацию существующих секций
