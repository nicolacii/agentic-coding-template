# Protocol: Prepare Prompt

## Context

- **Когда:** STANDARD/COMPLEX задачи, автоматически через core-master
- **Цель:** Улучшить промт для максимального качества с первого раза
- **Impact:** First-time Quality >80%, уменьшение переделок

| Сложность | Подготовка |
|-----------|------------|
| SIMPLE | Пропустить |
| STANDARD | Быстрая (30 сек) |
| COMPLEX | Полная |

**CRITICAL:**
- ПРАВИЛО #1: Прочитай ВСЮ инструкцию ПЕРЕД ответом
- ПРАВИЛО #2: Каждый ответ ДОЛЖЕН содержать RESPONSE FORMAT
- ПРАВИЛО #3: НЕ ПРОПУСКАЙ шаги — если пропустил, СТОП и вернись
- ПРАВИЛО #4: После выполнения каждого шага напиши коротко какой output сформировал от этого шага как самопроверка, что ты его не пропустил

---

## Requirements

### 1. Анализ исходного промта

```markdown
## PROMPT ANALYSIS

**Исходный:** "[промт пользователя]"

| Компонент | Найдено | Уточнить |
|-----------|---------|----------|
| **GOAL** | [есть/нет] | [что] |
| **CONTEXT** | [есть/нет] | [что] |
| **CONSTRAINTS** | [есть/нет] | [что] |
```

### 2. Извлечь EXPLICIT CONSTRAINTS

**CRITICAL:**
Выписать ДОСЛОВНЫЕ цитаты ограничений из запроса:

```markdown
### Explicit Constraints (цитаты):
- НЕ делать: "[цитата]"
- СДЕЛАТЬ: "[цитата]"
- КАК делать: "[цитата]"
```

Red Flags:
- "Пользователь имел в виду..." — интерпретация
- "Логично предположить..." — додумывание

WHY: AI скопировал .cursor внутрь git/, хотя пользователь сказал "скопирую 2 папки сам". Потеря 30 минут.

### 3. Улучшенный промт

```markdown
## IMPROVED PROMPT

**Goal:** [конкретный результат]
**Context:** [что известно, файлы]
**Constraints:** [что НЕ менять, ограничения]
**Success:** [как проверить готовность]
**Protocol:** [какой применить]
```

**REQUIRED:**
- Не интерпретировать — следовать буквально
- Выписать constraints дословно
- Выбрать протокол перед выполнением

---

## Examples

GOOD: Извлечение constraints
```markdown
**Исходный:** "Обнови правила, НЕ меняй имена файлов, сохрани весь контекст"

### Explicit Constraints:
- НЕ делать: "НЕ меняй имена файлов"
- СДЕЛАТЬ: "сохрани весь контекст"

**Improved:**
- Goal: Обновить правила по новому стандарту
- Constraints: Имена файлов НЕ МЕНЯТЬ, контекст сохранить
```

BAD: Интерпретация вместо следования
```
Пользователь просит обновить, значит можно переименовать для лучшей структуры...
```
Проблема: "НЕ меняй имена файлов" проигнорировано.

---

## Critical Points

**CRITICAL:**
- ДОСЛОВНО выписывать constraints из запроса
- НЕ интерпретировать — СЛЕДОВАТЬ
- Выбрать протокол ПЕРЕД выполнением
- При неясности — СПРОСИТЬ, не додумывать
